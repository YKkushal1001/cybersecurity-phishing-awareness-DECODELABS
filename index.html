What is Phishing?

Phishing is a social engineering attack where attackers impersonate legitimate entities to trick users into:

  • Revealing sensitive information (passwords, credit cards)
  • Clicking malicious links
  • Opening infected attachments
  • Installing malware
  • Transferring money
💡 Key Insight: Phishing succeeds by exploiting human psychology and trust, not necessarily complex technology.

Types of Phishing Attacks

  • Email Phishing: Most common type targeting large user bases with false links or attachments
  • Spear Phishing: Targeted attacks personalizing messages to specific individuals or departments
  • Whaling: High-value targets like CEOs or executives with authority to approve transactions
  • Clone Phishing: Creating legitimate-looking duplicates of known emails changing URLs or attachments
  • Vishing: Voice-based phishing using phone calls and VoIP to extract information
  • Smishing: SMS/text message phishing with malicious links or callback numbers

Common Attack Vectors

  • Domain Spoofing: Using domains resembling legitimate ones (amaz0n.com, appie.com)
  • URL Obfuscation: Hidden URLs using bit.ly, short links, or crafted anchor text
  • Homograph Attacks: Using visually identical Unicode characters (Cyrillic 'а' vs Latin 'a')
  • Subdomain Tricks: legitimate.domain.attacker.com appears legitimate at first glance
  • Attachment-Based: Executable files disguised as documents (.pdf.exe, .xls.scr)
  • Form Injection: Creating fake login forms within emails requesting credentials

Warning Signs

🚩 IMMEDIATE RED FLAGS:
• Unexpected urgency or threats
• Requests for passwords or personal data
• Suspicious sender addresses
• Unknown attachments or links
• Grammatical/spelling errors
• Generic greetings ("Dear Customer")
• Mismatched URLs and link text

Best Practices

  1. Verify Sender: Hover over sender name to see actual email address
  2. Check URLs: Hover over links (don't click!) to see the actual destination
  3. Look for Encryption: Check for HTTPS and security certificates
  4. Enable Multi-Factor: Use MFA on critical accounts for extra security
  5. Keep Software Updated: Regular updates patch security vulnerabilities
  6. Use Password Manager: Never autofill passwords on suspicious sites
  7. Report Suspicious Email: Use built-in report/spam buttons immediately
  8. Verify Out-of-Band: Call the organization directly using known numbers

If You Clicked a Link

  1. Disconnect immediately if asked for sensitive data
  2. Don't enter credentials if something feels wrong
  3. Scan your device with antivirus software
  4. Change passwords from a different, secure device
  5. Monitor accounts for unauthorized activity
  6. Contact IT support and report the incident
  7. Watch for identity theft signs for months afterward

📱 Tools & Services

  • VirusTotal: Scan suspicious files and URLs

  • Have I Been Pwned: Check if your data was breached
  • Google Safe Browsing: Built-in browser protection

🏢 Organizational Best Practices

  • Email Authentication: Implement SPF, DKIM, DMARC protocols
  • Security Training: Regular mandatory phishing awareness
  • Multi-Factor Auth: Require MFA on all critical systems
  • Security Tools: Email filtering, content scanning, threat intelligence
  • Incident Response: Have a clear reporting and escalation plan
  • Continuous Testing: Simulate phishing to assess vulnerability

⚠️ Report Phishing

  • To Your Organization: Forward to security@[company].com or IT Help Desk
  • To Email Provider: Gmail, Outlook, Yahoo all have report mechanisms
  • To Authorities: FBI IC3 (ic3.gov), CISA, local law enforcement

Key Takeaways

✓ Be Skeptical: Question unexpected emails, even from known senders
✓ Verify First: Call to confirm requests before taking action
✓ Hover, Don't Click: Preview links and sender addresses
✓ Report Immediately: Alert your IT team to threats
✓ Stay Updated: Keep software and security tools current
✓ Use MFA: Multi-factor authentication blocks most attacks
✓ Trust Your Instinct: When something feels wrong, it usually is
